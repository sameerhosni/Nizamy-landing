I am building a CRM/onboarding flow for Nizamy, a Saudi B2B HR SaaS product targeting SMEs in Saudi Arabia.

When a prospect fills out the Contact Us form on our website, they should automatically receive a bilingual welcome email (Arabic + English) that:


Confirms their submission
Communicates the Nizamy value proposition clearly
Includes a free trial / access link
Feels human and relationship-first — NOT like a generic autoresponder


There are 3 email templates to support. The active template is selected via an environment variable (no UI needed), so switching between templates is a one-line config change.


WHAT TO BUILD

1. Contact Form Handler (Backend API)

Build a POST endpoint at /api/contact that:


Accepts a JSON body:


json{
  "name": "string",
  "email": "string",
  "company": "string (optional)",
  "phone": "string (optional)"
}


Validates that name and email are present
Reads the active template from env var ACTIVE_TEMPLATE (values: "01", "02", "03")
Sends the correct bilingual HTML email via the existing Gmail SMTP / Nodemailer setup
Returns:


json{ "success": true, "message": "Email sent" }

or a proper error response if sending fails


2. The Three Email Templates

All templates are bilingual HTML emails — Arabic block first, English block below.
The recipient's name is injected dynamically. The TRIAL_LINK env var is injected as the CTA URL.

Shared email structure for all templates:


Subject line (bilingual)
Arabic body block
Visual divider
English body block
Single CTA button (bilingual label)
Footer with brand name

README — SETUP INSTRUCTIONS FOR THE USER

Include a README.md with these steps:

## Setup

1. In Replit Secrets (the lock icon), confirm the following are set:
   GMAIL_USER        → your Gmail address (already configured)
   GMAIL_APP_PASSWORD → your App Password (already configured)
   SENDER_NAME       → فريق نظامي | Nizamy Team
   ACTIVE_TEMPLATE   → 01  (or 02 or 03)
   TRIAL_LINK        → your free trial URL

2. Click Run. The server starts on port 3000.

3. To trigger the email, POST to /api/contact:
   {
     "name": "اسم العميل",
     "email": "client@example.com"
   }

4. To switch templates, change ACTIVE_TEMPLATE in Secrets and restart.


ADDITIONAL NOTES FOR REPLIT


Use process.env for all sensitive values. Never hardcode credentials.
The /api/contact endpoint should also log each submission to the console with timestamp, name, email, and which template was sent — useful for early-stage tracking before a CRM is in place.
Add a simple GET / route that returns { status: "Nizamy Email Service running" } as a health check.
Error handling: if the email fails to send, return HTTP 500 with { success: false, error: "Email delivery failed" } — do not expose the raw SMTP error to the client.
Gmail daily sending limit is 500 emails/day via App Password SMTP. This is sufficient for early GTM. Flag this limit in a comment in the code.
